## 2. 仍未完成（真正稳定运营前建议补齐）

### 2.1 异步任务架构（已具备基础版，仍建议增强）
- 现状：已提供 `scripts/task_worker.py`，可异步消费 `queued` 任务。
- 风险：当前是单进程轮询 Worker，尚未引入 Redis/Celery 级别的可靠投递与可视化管理。
- 建议：后续升级为 `Redis + Celery/RQ`。

### 2.2 鉴权体系（已上线 JWT）
- 现状：已支持 JWT `access + refresh`、refresh 轮换、`logout/logout-all` 撤销、过期清理脚本；`APP_ENV=prod` 下支持强制禁用调试通道与 mock token，且有启动安全校验。
- 建议：后续补 refresh token 风险设备指纹/多端策略（可选优化）。

### 2.3 运维与安全
- 现状：已具备结构化请求日志、`X-Request-Id`、基础限流（全局/auth/execute）、`/health/metrics` 指标接口；Sentry 已支持配置式初始化。
- 建议：
  - 日志聚合与告警（ELK/Loki/云日志）
  - Sentry 告警规则配置（生产环境 DSN 与告警通道）
  - DB 备份与恢复演练
  - 更细粒度风控策略（设备指纹/IP 黑白名单/行为规则）

### 2.4 审核回调闭环（已完成基础版）
- 现状：已提供腾讯回调接收接口，支持回调入库审计并联动作品自动下架。
- 建议：补回调验签（除 token 外）与重放防护（可选优化）。
