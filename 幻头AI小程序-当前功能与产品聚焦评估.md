# 幻头 AI 小程序：当前功能总览与产品聚焦评估

更新时间：`2026-03-16`

## 1. 当前整体判断

目前小程序已经不是“空壳 UI”，而是已经具备一条可以真实跑通的核心链路：

- 微信登录
- 选择创作入口
- 创建头像生成任务
- 异步生成中
- 查看结果
- 收藏 / 进入作品集
- 查看积分与任务记录

从“能不能跑起来”这个角度看，已经具备可演示、可继续打磨的基础。

但从“产品是否足够聚焦”这个角度看，确实已经开始出现偏移：

- 主线本来应该是“AI 头像生成”
- 现在部分页面和交互开始向“通用修图工具”靠拢
- 尤其是 `quick-edit` 和 `image-edit` 里的部分表达，会让产品看起来像轻量修图软件，而不是头像生成产品

一句话判断：

当前版本已经具备“头像生成产品雏形”，但需要主动收口，避免继续长成一个泛化图片工具。

---

## 2. 已经实现的功能

### 2.1 核心头像生成链路

这一部分已经基本成型，并且大多数环节已经接上真实后端：

- `login`：真实微信登录入口
- `home`：首页，展示灵感入口和最近作品
- `create-menu`：创作工坊，作为生成入口总页
- `text-generate`：一句话生成头像
- `image-reference`：参考图 / 风格迁移生成头像
- `image-edit`：基于已有头像做增强或二次生成
- `loading`：真实异步任务轮询，展示 `queued / running / succeeded / failed`
- `result`：展示生成结果、多张候选图、收藏入口

这条链路的关键能力已经具备：

- 文生图已接真实 backend
- 风格迁移已接真实 backend
- 图生图已接真实 backend
- 火山 API 已接入
- 支持单图 / 多图候选输出
- 任务已改成真实异步队列
- 生成结果已写入作品库

### 2.2 作品与个人中心

这部分已经达到“可用”状态：

- `assets`：我的作品 / 收藏 两个板块
- `history`：创作记录
- `profile`：个人中心

已实现能力：

- 查看自己生成过的作品
- 查看收藏
- 查看任务历史
- 从结果页收藏作品

### 2.3 积分体系

这部分也已经不是展示稿，而是真实逻辑：

- `points`：会员中心 / 积分页
- 每日签到加积分
- 激活码兑换积分
- 查看积分流水
- 查看当前余额
- 生成任务扣积分

当前已实现的积分能力：

- 登录奖励
- 每日签到
- 激活码兑换
- 任务创建扣分
- 失败返还

### 2.4 后端与资源访问层

这部分虽然用户不直接看到，但对上线非常关键，目前已经处理到相对稳定：

- 微信登录态
- token 刷新
- 真机请求真实后端
- 任务异步 worker
- COS 存储
- COS 签名图链接返回
- 历史 mock 图从作品列表中清理

## 3. 现在已经偏离主线的部分

这里是我认为最需要你做产品判断的地方。

### 3.1 `quick-edit` 页面偏离明显

这个页面现在更像“参数式图片编辑器”：

- 旋转
- 亮度
- 饱和度
- 对比度
- 输出格式
- 输出质量

这套能力更像通用修图工具，和“做一张好看的头像”之间关系不大。

判断：

- 研发上它是能存在的
- 但产品上不应该放进头像生成主流程
- 它会稀释“幻头 = 头像生成”的认知

建议：

- `quick-edit` 不作为主入口
- 后续可以隐藏、下线，或只留给内部调试

### 3.2 `image-edit` 有一部分已经开始泛修图化

`image-edit` 目前的表达里包含：

- AI 换背景
- 贴纸库
- 局部重绘
- 高级工具

问题不在于这些功能不能做，而在于它们会把产品从“头像生成”带向“修图工作台”。

其中：

- “AI 换背景”仍然和头像场景有关
- “局部重绘”如果限定在头像，也可以成立
- “贴纸库”容易变成泛娱乐修图
- “高级工具”最容易让产品失焦

建议：

- 保留“头像增强”“背景优化”“风格统一”
- 弱化“贴纸库”“高级工具”这些修图软件味道很重的表达

### 3.3 `service` 页面仍然是 demo 内容页

`service` 目前还是通过 demo 数据驱动的服务说明页，更像占位页面，不是核心产品能力。

它的问题不是“做错了”，而是：

- 当前不是最优先
- 它不影响头像生成闭环
- 它会占用注意力

建议：

- 先维持最基础的帮助 / 关于
- 不要在这一阶段投入太多精力

### 3.4 首页表达还没有完全聚焦“头像”

首页现在已经比早期更接近产品化，但还存在两个问题：

- 有一部分是“灵感 / 推荐 / 作品流”的表达
- 还没有足够直接地把用户推向“生成头像”

建议首页应该更像：

- 一句话生成头像
- 上传自拍生成头像
- 热门风格模板
- 最近生成记录

而不是把首页做成偏社区、偏浏览的信息流首页。

## 4. 当前真正适合保留的产品主线

如果我们把“幻头”重新收口成一个清晰产品，我建议主线就是下面这条：

### 主线 A：一句话生成头像

- 输入提示词
- 选择风格
- 选择候选图数量
- 生成头像

### 主线 B：上传自拍生成头像

- 上传原图
- 选择风格模板
- 生成头像

### 主线 C：已有头像做轻量增强

- 上传已有头像
- 换背景
- 提升质感
- 做统一风格

### 主线 D：结果沉淀

- 收藏
- 进入作品集
- 再次生成

这四条已经足够支撑一个明确的头像产品。

---

## 5. 建议从“主做”变成“暂缓 / 降权”的内容

### 应该主做

- 文生图头像
- 自拍转头像
- 风格迁移头像
- 候选图多图输出
- 生成中状态体验
- 结果页与作品集
- 积分体系

### 应该降权

- 通用快速编辑
- 亮度 / 饱和度 / 对比度这类参数调节
- 贴纸库
- 泛化局部重绘
- 类修图软件的“高级工具”
- 过强的社区 / 广场表达

### 应该后置

- 会员支付
- 公共广场
- 复杂本地编辑器
- 贴纸资产运营
- 完整服务说明体系

## 6. 以“头像生成产品”视角看，当前页面建议分层

### 第一优先级：必须保留

- `login`
- `home`
- `create-menu`
- `text-generate`
- `image-reference`
- `image-edit`
- `loading`
- `result`
- `assets`
- `profile`
- `points`

### 第二优先级：保留但弱化

- `history`

它是有价值的，但不应该抢主线入口。

### 第三优先级：暂缓 / 隐藏

- `quick-edit`
- `service`

它们可以保留代码，但不应该继续作为产品重点。

## 7. 我对当前版本的产品结论

### 已经具备的产品能力

- 已经有真实的头像生成链路
- 已经有结果沉淀和积分体系
- 已经可以做一版“可用的头像生成小程序”

### 当前最大风险

- 功能继续发散
- 页面命名和表达继续往修图软件方向走
- 导致用户不知道“这个产品到底是生成头像，还是做图片编辑”

### 最重要的收口建议

后续所有功能判断，都先问一句：

这个功能是不是在帮助用户“更快获得一张满意的头像”？

如果答案不是很明确，就不应该进入当前主版本。

---

## 8. 我建议的 V1 产品定义

如果现在让我帮你把产品重新定义成一句话，我会建议写成：

**幻头是一款专注于 AI 头像生成的小程序，用户可以通过一句话、自拍上传或风格参考图，快速获得多张可选头像结果，并沉淀到个人作品集。**

这个定义下：

- 文生图是核心
- 自拍转绘是核心
- 风格迁移是核心
- 轻量增强可以有
- 通用修图不要成为重点

---

## 9. 下一步建议

我建议后续开发按这个顺序收口：

1. 把首页、创作工坊、结果页的文案和入口全部统一成“头像生成”
2. 把 `quick-edit` 从主产品视角降权
3. 重新定义 `image-edit`，只保留和头像直接相关的增强能力
4. 首页不再往社区流扩，先把“头像生成主线”做强
5. 作品页补“再次生成 / 下载 / 删除 / 设为收藏”

---

## 10. 这份文档的结论

当前项目没有跑偏到不可收拾，主骨架仍然是头像生成。

但如果现在不主动收口，后面确实会越来越像“轻量修图软件”。

所以现在最正确的方向不是继续加更多功能，而是：

**把已经做出来的能力重新组织成一个更聚焦的 AI 头像生成产品。**
