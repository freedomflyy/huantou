# 幻头 AI头像生成小程序 - 详细开发计划

版本：`v1.0`  
最后更新：`2026-03-12`  
文档用途：作为开发执行基线，记录当前进度、后续排期、Mock 去除计划与上线方案。

---

## 1. 项目目标与范围

### 1.1 目标
- 上线一款国内可用的微信小程序，支持 AI 头像创作与基础编辑。
- 首版并发目标：`20`。
- 图片保留策略：`7天`。
- 积分系统先落地，不接微信支付。

### 1.2 首版功能范围（MVP）
- 文生图（txt2img）
- 图生图修改（img2img）
- 双图风格迁移（style transfer）
- 快速编辑（裁剪、旋转、亮度/对比/饱和、贴纸、背景替换入口）
- 结果预览、历史记录、资产管理
- 后台管理（首版）：用户管理、积分调整、任务重试、作品下架

## 2. 技术选型（当前决策）

### 2.1 前端（小程序）
- 技术：微信原生 `WXML + WXSS + JS/TS`
- 说明：当前 `ui/` 原型已联通，后续按页面迁移到小程序目录。
- 原因：迁移路径最短、风险最低、便于快速上线。

### 2.2 后端
- 技术：`FastAPI + SQLAlchemy + PostgreSQL`
- 当前数据库：本地 PostgreSQL（已安装、可用）
- 说明：AI 生成必须走异步任务模型，避免同步阻塞。

### 2.3 AI 生成能力层
- 设计：Provider 适配层
  - `mock_provider`（当前联调）
  - `local_comfyui_provider`（本地 4070）
  - `volcengine_provider`（后续替换/扩容）
- 原则：API 契约先稳定，再替换执行后端。

### 2.4 存储与审核
- 图片存储：短期 `mock URL/本地`，上线前切 COS（或同级对象存储）
- 内容审核：当前 `mock/预留接口`，上线前接真实审核服务

## 5. 分阶段计划（含时间安排）

> 以 `2026-03-13` 为正式开发第 1 天，按 1 周一个阶段滚动交付。

### Phase 1（2026-03-13 ~ 2026-03-19）基础联调阶段
- 前端：
  - 建立小程序工程结构（页面、路由、全局样式 token）
  - 迁移首页、创建页、文本生成页、结果页、预览页
  - 接通登录与积分余额展示
- 后端：
  - 完成 API 错误码规范与响应封装
  - 增加任务状态流转接口（queued/running/succeeded/failed）
  - 补充 API 文档（OpenAPI 示例请求）
- 验收：
  - 小程序可完成：登录 -> 创建任务 -> 查看任务 -> 查看积分

### Phase 2（2026-03-20 ~ 2026-03-26）功能完善阶段
- 前端：
  - 迁移 image-edit、image-reference、quick-edit、history、assets、profile
  - 快速编辑模块先实现：裁剪/旋转/亮度/对比/饱和
- 后端：
  - 新增作品管理 API（列表、删除/下架标记）
  - 新增后台管理 API 草版（用户查询、积分调整、任务重试）
  - 增加审计日志表与操作日志落库
- 验收：
  - 全主流程在小程序端可走通（仍可使用 Mock 生成）

### Phase 3（2026-03-27 ~ 2026-04-02）登录与安全接入阶段
- 去除 Mock：
  - 微信登录改真实 `code2Session`
- 新增：
  - 内容审核 API 接口协议定稿（仍可挂 mock provider）
  - 请求签名、基础限流、IP/用户维度风控
- 验收：
  - 登录链路从 Mock 切真实，兼容旧测试用户数据

### Phase 4（2026-04-03 ~ 2026-04-09）AI 执行真实化阶段
- 后端：
  - 引入任务队列（建议 Redis + Celery/RQ）
  - Provider Adapter 实装：
    - `local_comfyui_provider` 或 `volcengine_provider` 至少上线 1 个真实执行链路
  - mock-complete/mock-fail 仅保留测试开关
- 存储：
  - 接入 COS 上传与签名 URL
  - `assets` 改为真实对象存储地址
- 验收：
  - 至少一条生成链路“真实出图”

### Phase 5（2026-04-10 ~ 2026-04-16）预发布阶段
- 管理后台（首版）：
  - 用户管理、积分调整、任务重试、作品下架
- 运维：
  - 部署 staging 环境
  - 日志、告警、备份策略落地
  - 图片 7 天清理任务（定时任务）
- 审核：
  - 接入真实审核服务（文本/图片）
- 验收：
  - 预发布环境端到端稳定运行，核心路径无阻断

### Phase 6（2026-04-17 ~ 2026-04-23）上线准备与灰度
- 小程序提交审核与灰度
- 上线前 Checklist 完成（见第 8 章）
- 灰度指标观测（错误率、任务成功率、平均时延、投诉率）
- 验收：
  - 国内正式发布（首批用户）

## 8. 部署与上线方案

### 8.1 环境分层
- `dev`：本地开发（可内网映射）
- `staging`：云端预发布（建议）
- `prod`：正式发布环境

### 8.2 部署组件（目标形态）
- API 服务：FastAPI
- DB：PostgreSQL
- Queue：Redis
- Worker：AI 任务执行器（ComfyUI/火山引擎适配）
- Object Storage：COS
- 监控：Sentry + 基础服务监控

### 8.3 上线前 Checklist
- 微信合法域名、HTTPS、备案完成
- 登录、生成、积分、历史、资产全链路可用
- 审核策略与拦截规则上线
- 数据备份与恢复演练完成
- 日志与告警可用
- 7天图片过期清理任务生效

## 9. 风险与应对

- 风险：本地 GPU 性能不足或不稳定  
  应对：Provider 抽象 + 火山引擎可切换

- 风险：Mock 去除过晚导致集成延期  
  应对：按 Phase 3/4 强制里程碑切换

- 风险：审核未接入导致提审风险  
  应对：Phase 5 前必须接真实审核服务

- 风险：数据模型后期频繁变更  
  应对：尽快引入迁移工具（Alembic）并执行版本化迁移
